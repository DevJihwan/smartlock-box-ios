# 씽크프리 (ThinkFree) - iOS 앱 요구사항정의서 (심플 버전)

## 📋 프로젝트 개요
- **프로젝트명**: 씽크프리 (ThinkFree)
- **플랫폼**: iOS
- **개발언어**: Swift
- **버전**: v2.0 (Simplified)
- **문서 작성일**: 2025년 11월 13일

## 🎯 배경 및 목적

### 배경
현대 사회에서 스마트폰 과다 사용이 일상생활과 업무 집중력을 방해하고 있습니다. 특히 특정 시간대(업무 시간, 공부 시간 등)에 스마트폰 사용을 제한하고자 하는 니즈가 증가하고 있습니다.

### 목적
사용자가 지정한 시간대에 사용 제한을 설정하고, 제한 시간 초과 시 AI 기반 창의력 평가를 통해서만 해제할 수 있는 **초심플** 디지털 디톡스 앱을 제공합니다.

## 🔧 핵심 기능 (단 하나!)

### 시간대별 스크린 제어

#### 기능 설명
사용자가 특정 시간대(예: 오전 9시 ~ 오후 3시)를 설정하고, 해당 시간대 내에서 사용 가능한 총 시간(예: 2시간)을 제한합니다. 설정된 시간을 모두 사용하면 스크린이 잠기며, AI가 인정하는 창의적인 문장을 작성해야만 해제할 수 있습니다.

#### 설정 요소
1. **시간대 설정**
   - 시작 시간 (예: 오전 9시)
   - 종료 시간 (예: 오후 3시)

2. **제한 시간 설정**
   - 해당 시간대 내 최대 사용 가능 시간 (예: 2시간)

3. **활성화/비활성화**
   - 간단한 토글 스위치로 ON/OFF

#### 작동 방식

**예시 시나리오:**
```
설정: 오전 9시 ~ 오후 3시, 제한 시간 2시간

오전 9:00 - 제어 시작
오전 9:00 ~ 11:00 - 스마트폰 사용 (2시간)
오전 11:00 - 제한 시간 도달, 스크린 잠금!
오전 11:00 ~ 오후 3:00 - 잠금 상태 유지 (창의적 해제만 가능)
오후 3:00 - 설정된 시간대 종료, 자동 해제
```

## 🛠 기술 요구사항

### iOS 개발 환경
- **최소 지원 버전**: iOS 15.0+
- **개발 도구**: Xcode 14+
- **언어**: Swift 5.5+
- **아키텍처**: MVVM 패턴
- **UI**: SwiftUI

### 주요 기술 스택
- **Screen Time API**: 앱 사용 시간 모니터링
- **UserDefaults**: 설정 및 상태 저장
- **URLSession**: API 통신
- **UserNotifications**: 잠금 알림
- **LocalAuthentication**: Face ID/Touch ID (선택사항)

### 외부 API
- **OpenAI GPT-4 API**: 창의력 평가
- **Anthropic Claude API**: 이중 검증

### 데이터 관리
- **단어 데이터베이스**:
  - 한국어: 20,000개 단어
  - 영어: 20,000개 단어
  - JSON 포맷으로 앱 번들에 포함
- **로컬 저장**:
  - 시간대 설정
  - 제한 시간
  - 일일 사용 기록
  - API 키
  - 언어 설정

## 📋 사용자 스토리

### Story 1: 시간대 제어 설정
```
직장인으로서 업무 시간(오전 9시~오후 6시)에 스마트폰 사용을 2시간으로 제한하고 싶다.
그래서 앱에서 해당 시간대와 제한 시간을 설정하면, 2시간 사용 후 자동으로 잠긴다.
```

### Story 2: 창의적 해제
```
급한 업무가 생겨 잠금을 해제해야 한다.
그래서 "구름"과 "자유" 단어로 창의적인 문장을 만들어 AI 승인을 받아 해제한다.
```

### Story 3: 자동 해제
```
설정한 시간대가 끝나면 자동으로 제어가 해제되어 정상적으로 스마트폰을 사용할 수 있다.
```

### Story 4: 응급 상황
```
잠금 상태에서도 전화와 문자는 항상 사용할 수 있어 응급 상황에 대비할 수 있다.
```

### Story 5: 언어 전환
```
영어권 사용자로서 영어로 앱을 사용하고 싶다.
그래서 설정에서 English를 선택하면 모든 화면이 영어로 전환된다.
```

## ⚠️ 제약사항 및 고려사항

### 기술적 제약
- iOS Screen Time API의 권한 요청 필요
- 시스템 레벨 잠금의 한계 (완전한 차단은 불가능)
- API 키 필수 (사용자 직접 입력)

### 보안 및 개인정보
- API 키는 Keychain에 안전하게 저장
- 사용자 데이터 로컬 저장 원칙
- 창의적 답변은 API 전송 후 즉시 삭제
- 외부 서버에 데이터 전송 없음

### 사용성
- 하루 10회 제출 제한으로 과도한 시도 방지
- 하루 3회 단어 변경으로 너무 쉬운 해제 방지
- 응급상황 대비 전화/문자 항상 허용
- 설정된 시간대 종료 시 자동 해제

### API 비용
- 사용자가 직접 API 키 입력
- 앱은 API 비용 청구하지 않음
- 사용량은 사용자 책임
